Cross-Site Scripting (XSS)

One of the most common and dangerous web vulnerabilities

What is XSS?

Cross-Site Scripting (XSS) is a security vulnerability that allows attackers to inject malicious client-side scripts (usually JavaScript) into web pages viewed by other users. When the victim's browser executes the injected script, the attacker can steal cookies, session tokens, sensitive data, or perform actions on behalf of the victim.

Types of XSS

  • Reflected XSS: Malicious script is reflected off a web server (e.g., via URL parameters). The script is not stored.
  • Stored XSS: Malicious script is permanently stored on the server (in a database, comment, etc.) and served to users.
  • DOM-based XSS: The vulnerability exists in client-side code (JavaScript) rather than server-side code. The payload never reaches the server.

History of XSS

The concept of XSS was first publicly discussed in the late 1990s. The term "Cross-Site Scripting" was coined around 2000. One of the earliest documented cases was in 1999 when a vulnerability in Hotmail allowed attackers to steal session cookies.

In the early 2000s, XSS became one of the most prevalent web vulnerabilities as dynamic web applications grew rapidly. The Open Web Application Security Project (OWASP) has consistently ranked XSS in its Top 10 web vulnerabilities for many years.

Major XSS Incidents

2005 – MySpace Samy Worm

One of the most famous XSS attacks in history. Samy Kamkar created a worm that spread via MySpace profiles. Within hours, over one million users were infected. The worm added "Samy is my hero" to victims' profiles and stole their session cookies.

2008 – Twitter XSS Attack

Attackers exploited a reflected XSS vulnerability on Twitter, allowing them to post tweets on behalf of other users.

2010 – PayPal Persistent XSS

A stored XSS vulnerability allowed attackers to inject malicious JavaScript that stole user credentials.

2014 – eBay XSS Vulnerability

Multiple XSS flaws were discovered that could have allowed attackers to steal session information from millions of users.

2018 – British Airways Data Breach

Attackers used Magecart-style skimming scripts (often delivered via XSS or supply chain attacks) to steal payment card details from the checkout page. Over 380,000 payment cards were compromised.

2020s – Ongoing Supply Chain & DOM XSS Attacks

Modern attacks frequently target third-party libraries, advertisements, and complex JavaScript frameworks. DOM-based XSS has become increasingly common due to heavy client-side rendering in modern web apps.

Impact of XSS Attacks

  • Account takeover
  • Session hijacking
  • Stealing sensitive user data
  • Defacement of websites
  • Spreading malware or worms
  • Keylogging and form grabbing
  • Redirecting users to phishing sites

Prevention

Modern web frameworks provide built-in protection, but developers must still be careful with user input. Key defenses include:

  • Input validation & output encoding
  • Content Security Policy (CSP)
  • Using secure frameworks with auto-escaping
  • HTTPOnly and Secure cookies
  • Subresource Integrity (SRI) for third-party scripts
